# zc-sporttery-hub

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态 HTML 页面的集中存放与版本管理。

## 项目简介

`zc-sporttery-hub` 旨在为多个独立的 HTML 页面提供统一的归档与分发空间。每个页面均为独立、自包含的 HTML 文件，可单独访问，无需依赖后端服务。仓库结构简洁，便于维护与扩展。

## 目录说明

```
zc-sporttery-hub/
├── pages/          # 存放独立 HTML 页面文件
├── assets/         # 公共资源（CSS、JS、图片等，可选）
├── index.html      # 仓库首页（可选）
└── README.md       # 本文件
```

- `pages/`：所有独立 HTML 页面按需存放于此，可进一步按子目录分类。
- `assets/`：若页面间共享样式、脚本或图片，可放置于此，但建议每个页面尽量保持自包含。
- `index.html`：可选的仓库入口页面，用于展示页面列表或导航。

## 页面归档说明

- 每个 HTML 页面应为**独立完整**的文件，包含必要的样式与脚本。
- 页面之间无强制依赖关系，可独立部署与访问。
- 归档时请保持文件名清晰、有含义，避免特殊字符。
- 鼓励在页面内部添加版本注释或修改日期，便于追溯。

## 维护说明

- 本仓库接受新增、更新或移除页面的 Pull Request。
- 新增页面请放入 `pages/` 目录，并确保文件可正常渲染。
- 更新页面时请保留旧版本历史（通过 Git 记录），不建议直接覆盖重要文件。
- 维护者会不定期检查页面可用性，但不对页面内容的准确性或时效性负责。

## 使用建议

- 可通过 GitHub Pages 或其他静态托管服务直接部署本仓库，实现页面在线访问。
- 如需引用仓库中的页面，建议使用相对路径或基于部署后的公开 URL。
- 如有特殊需求（如跨域资源、外部 API 调用），请确保页面自行处理。

## 贡献指南

欢迎提交 Issue 或 Pull Request。提交前请确认：
- 页面无恶意代码、无违规内容。
- 页面文件命名合理，不与其他页面冲突。
- 若涉及公共资源修改，请同步更新相关引用。
